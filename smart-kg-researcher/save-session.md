# 세션 → 지식 그래프 저장

현재 세션의 대화 내용을 분석하여 MCP 지식 그래프와 `~/knowledge/` 파일에 저장합니다.

---

## 절차

### 1. 대화 내용 분석

현재 세션 전체를 검토하여 다음을 파악합니다:

- 어떤 기술/개념이 다뤄졌는가?
- 새롭게 학습되거나 명확해진 사실이 있는가?
- 기존 KG 엔티티에 추가할 observations가 있는가?
- 파일로 보존할 만큼 충분한 내용인가?

### 2. 기존 KG와 비교

MCP `search_nodes`로 각 핵심 주제가 이미 존재하는지 확인합니다.

**기존 엔티티가 있는 경우:**
- MCP `add_observations` 도구로 새 정보만 추가합니다.
- 기존 엔티티를 삭제하거나 재생성하지 않습니다.

**새 엔티티인 경우:**
- MCP `create_entities` 도구로 생성합니다.
- 엔티티 간 관계가 있으면 `create_relations`도 호출합니다.

### 3. 저장 대상 판단

| 조건 | 행동 |
|---|---|
| 완전히 새로운 주제 (KG + 파일 모두 없음) | 새 `.md` 파일 생성 + KG 엔티티 생성 |
| 기존 파일이 있는 주제, 추가 내용 있음 | 기존 파일에 내용 추가(append) + KG observations 추가 |
| 기존 KG 엔티티만 있고, 추가 내용이 적음 | 파일 생략, KG observations만 추가 |
| 단순 Q&A, 짧은 확인성 대화 | 파일 저장 생략, KG만 업데이트 |

### 4. 파일 저장 (필요한 경우)

**저장 경로:** `~/knowledge/research/<주제명>/` 또는 기존 파일 경로

**파일 형식 (frontmatter 필수):**
```markdown
---
title: "주제 제목"
tags: [tag1, tag2]
related_nodes: []
last_synced_with_graph: ""
session_date: "YYYY-MM-DD"
---

# 제목

## 내용
...
```

**파일 작성 원칙:**
- 대화에서 나온 핵심 내용만 추출, 대화체 제거
- 기술적 사실, 예시 코드, 비교표 우선 보존
- 개인적 맥락(운영 중인 테이블명 등)은 별도 섹션으로 구분

### 5. KG 업데이트

새 파일이 저장됐거나 새 엔티티가 생성된 경우:
- 파일의 frontmatter `related_nodes`, `last_synced_with_graph` 업데이트
- Write 도구로 파일 저장

### 6. 완료 메시지 출력

```
저장 완료:
- KG 엔티티 추가/업데이트: N개
- 파일 저장: ~/knowledge/research/...  (없으면 "파일 저장 생략")
```

---

## 실패 처리

어느 단계에서든 오류 발생 시:
- 오류 내용을 사용자에게 알립니다.
- 이미 저장된 파일은 그대로 유지합니다.
- KG 부분 업데이트는 롤백하지 않습니다.
