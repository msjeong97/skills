# Phase 3: 웹 리서치 및 파일 저장

## 리서치 제안

사용자에게 다음 메시지를 출력합니다:
> "현재 리서치 파일과 지식 그래프에 **[질문 주제]**에 대한 정보가 부족합니다.
> 웹 리서치를 수행하고 결과를 `~/knowledge/`에 저장할까요?"

**사용자가 거절한 경우:**
"정보가 부족합니다."로 시작하는 답변과 함께 부분적으로 찾은 내용을 제공합니다. Phase 4는 수행하지 않습니다.

**사용자가 동의한 경우:** 아래 절차를 수행합니다.

---

## 웹 리서치

1. WebSearch 도구로 핵심 키워드 검색 (2~3회, 다른 키워드 조합 사용)
2. 상위 결과 URL 2~3개를 WebFetch 도구로 내용 수집
3. 핵심 내용을 마크다운으로 정리합니다

---

## 저장 경로 결정

1. `~/knowledge/research/` 하위 폴더를 확인합니다 (Bash: `ls ~/knowledge/research/`)
2. 주제와 가장 관련 있는 폴더를 선택합니다. 없으면 새 폴더명을 결정합니다.
3. 해당 폴더의 `.md` 파일 개수를 셉니다 (**쓰기 전** 카운트):
   ```bash
   ls ~/knowledge/research/{topic}/*.md 2>/dev/null | wc -l
   ```
4. `index = 파일 수 + 1` (zero-padding: 01, 02, ... 09, 10, ...)
5. `slug` = 제목의 첫 4~6 단어를 소문자+하이픈으로 변환
   - 예: `"Apache Kafka 심층 분석"` → `apache-kafka-심층-분석`
   - 한글 단어는 그대로 포함합니다
6. 최종 경로: `~/knowledge/research/{topic}/{index}-{slug}.md`
   - 필요하면 폴더를 먼저 생성합니다: `mkdir -p ~/knowledge/research/{topic}`

---

## 파일 저장

아래 구조로 Write 도구를 사용하여 파일을 저장합니다:

```
---
title: "[리서치 주제명]"
tags: [tag1, tag2]
related_nodes: []
last_synced_with_graph: ""
---

[리서치 내용 — 마크다운 형식]

<!-- 3개 이상의 컴포넌트/단계가 있는 흐름 또는 계층 구조가 있을 때만 Mermaid 포함 -->
```

저장 완료 후:
- 저장된 파일의 전체 경로를 사용자에게 알립니다.
- `refine.md` (Phase 4)를 Read 도구로 읽어 그래프 업데이트를 진행합니다.
